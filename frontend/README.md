# Платёжный календарь — Frontend

React-фронтенд для платёжного календаря. Работает с Laravel API из папки `backend/`.

## Требования

- Docker Desktop;
- Node.js 18+;
- npm или pnpm;
- запущенный backend на `http://localhost:8000/api`.

## Запуск backend

Из корневой папки проекта:

```bash
docker compose up -d backend
```

Backend:

```text
http://localhost:8000
```

Первый запуск может занять несколько минут: Docker скачает образы, ставит зависимости, создаёт `.env`, генерирует `APP_KEY` и запускает миграции.

## Запуск frontend

Из папки `frontend/`:

```bash
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

В Windows PowerShell лучше использовать `npm.cmd`:

```bash
npm.cmd install
npm.cmd run dev -- --host 127.0.0.1 --port 5173
```

Фронт:

```text
http://127.0.0.1:5173
```

Не закрывайте терминал с Vite, пока тестируете интерфейс.

## Что реализовано

- вход и авторизация через API;
- работа с Bearer token;
- отображение роли пользователя;
- платёжный календарь с сеткой по дням и счетам;
- три режима календаря: неделя, месяц, квартал;
- подсветка кассовых разрывов (отрицательный остаток);
- перенос платежа на другую дату через drag-and-drop;
- фильтры по счёту, статье, контрагенту;
- создание заявок на платёж;
- статусы заявок: черновик, на согласовании, согласована, в реестре, оплачена, отклонена;
- многоэтапное согласование с маршрутами;
- выбор маршрута при отправке на согласование;
- журнал согласований с комментариями;
- повторяющиеся платежи (шаблоны);
- создание/редактирование/удаление шаблонов;
- пауза/возобновление шаблонов;
- генерация черновика из шаблона;
- плановые поступления;
- подтверждение и получение поступлений;
- приоритеты (высокий, средний, низкий);
- формирование реестра платежей;
- оплата реестра;
- экспорт реестра в CSV;
- мультивалютность (RUB, USD, EUR, CNY);
- конвертация валют по курсу;
- вкладка «Валюты» в справочниках;
- отчёты: остатки по счетам, кассовые разрывы, план и факт;
- навигатор по месяцам в отчёте «План и факт»;
- экспорт отчётов в CSV;
- справочники: счета, контрагенты, статьи, пользователи;
- журнал аудита действий;
- 4 темы оформления (Шалфей, Мальва, Коралл, Изумруд);
- профиль пользователя;
- настройки (уведомления, часовой пояс, плотность таблиц);
- обработка загрузки, пустых состояний и API-ошибок.

## Основной сценарий проверки

1. Войти за Инициатора (Иванова М.С.).
2. Открыть «Заявки на платёж» → нажать «Новая заявка».
3. Заполнить форму: сумма, дата, счёт, контрагент, статья.
4. Выбрать маршрут согласования (Стандартный).
5. Нажать «Отправить на согласование».
6. Проверить что статус изменился на «На согласовании».
7. Проверить что появилась кнопка маршрута (🔀) и точки MiniStepper.
8. Войти за Руководителя (Козлова Е.В.).
9. Открыть «Заявки» → нажать 🔀 на заявке.
10. В Drawer нажать «Согласовать» с комментарием.
11. Проверить что этап 1 стал «Согласовано», этап 2 стал активным.
12. Войти за Казначея (Петров И.А.).
13. Открыть «Календарь» → проверить что кассовые разрывы подсвечены.
14. Перетащить разрыв на другой день (drag-and-drop).
15. Проверить что остатки пересчитались.
16. Открыть «Реестр платежей» → сформировать реестр.
17. Оплатить реестр → проверить что статус изменился.
18. Выгрузить реестр в CSV.
19. Открыть «Отчёты» → проверить вкладки «Остатки», «Кассовые разрывы», «План и факт».
20. Сменить тему в Настройках → проверить что цвета изменились.

## API

Base URL:

```text
http://localhost:8000/api
```

Общие заголовки:

```text
Accept: application/json
Authorization: Bearer <token>
```

Авторизация:

- `POST /api/login`;
- `POST /api/logout`;
- `GET /api/me`.

Справочники:

- `GET/POST /api/accounts`;
- `GET/POST /api/counterparties`;
- `GET/POST /api/items`;
- `GET/POST /api/users`.

Заявки:

- `GET/POST /api/payments`;
- `PUT/DELETE /api/payments/{id}`;
- `POST /api/payments/{id}/submit`;
- `POST /api/payments/{id}/approve`;
- `POST /api/payments/{id}/reject`;
- `POST /api/payments/{id}/move`.

Поступления:

- `GET/POST /api/incomes`;
- `PUT/DELETE /api/incomes/{id}`;
- `POST /api/incomes/{id}/confirmed`;
- `POST /api/incomes/{id}/received`.

Календарь:

- `GET /api/calendar?start_date=...&end_date=...`.

Реестры:

- `GET/POST /api/registries`;
- `GET /api/registries/{id}`;
- `POST /api/registries/{id}/pay`;
- `GET /api/registries/{id}/export`.

Отчёты:

- `GET /api/reports/balances`;
- `GET /api/reports/cash-gaps`;
- `GET /api/reports/plan-fact`.

## Формат данных

Backend отдаёт суммы в копейках (целое число). Фронтенд конвертирует в рубли только при отображении через функцию `kopecksToRub()`. При отправке на бэкенд — обратно через `rubToKopecks()`.

Маппинг полей:

- `account_name` → столбец «Счёт»;
- `counterparty` → столбец «Контрагент»;
- `item` → столбец «Статья»;
- `planned_date` → столбец «Дата»;
- `amount` → столбец «Сумма» (конвертируется из копеек);
- `priority` → столбец «Приоритет»;
- `status` → столбец «Статус».
